Oracle - SQL 설치(Windows)

2022-04-25

#SQL, Oracle, Windows

SQL Oracle 설치(Windows)

  • 오라클 설치 과정을 기록한다.
  • 설치 파일
    • Oracle 링크: https://www.oracle.com/kr/database/technologies/oracle19c-windows-downloads.html

Oracle 설치

  1. 설치파일의 압축을 해제한 후, 폴더 안의 Setup파일을 관리자 권한으로 실행한다.

  2. 단일 인스턴스 데이터베이스 생성 및 구성을 선택한다.

  3. 데스크톱 클래스를 선택한다.

  4. 가상 계정 사용을 선택한다.

  5. 구성 값을 설정한다.

    • 만약 비밀번호 보안 문제가 발생한다면, 비밀번호를 복잡한 것으로 변경하거나 다른 설정값으로 진행한다.(아래에 추가 서술)

  6. 설치를 진행한다.

비밀번호 보안 문제 발생 시

  1. 소프트웨어만 설정 을 선택한다.

    • 이 방식대로 진행 시, 전역 데이터 베이스가 없기 때문에 DB 생성을 따로 진행해 주어야 한다.

  2. 단일 인스턴스 데이터베이스 설치 를 선택한다.

  3. Enterprise Edition 설치 를 선택한다.

  4. 가상 계정 사용 선택 을 선택한다.

  5. 경로를 지정한다.

  • 이후는 동일하다.

데이터 베이스 생성

  1. 터미널을 실행한다.

  2. SQL 설치파일이 있는 폴더로 들어간다.

  3. dbca 명령어를 실행한다.

  4. 데이터 베이스 생성을 선택한다.

  5. 설정값을 추가한다.

    • 데이터베이스 이름, 저장 영역 유형, 비밀번호 설정만 수정하였다.

오라클 설치 완료 후 설정

  1. SQL Plus 를 실행한다.

  2. 로그인한다.(system / 비밀번호)

  3. 테이블 스페이스를 생성한다.

    CREATE TABLESPACE myts DATAFILE 'C:\sql_lecture\oradata\MYORACLE\myts.dbf' SIZE 100M AUTOEXTEND ON NEXT 5M;

    • C와 oradata 사이의 경로만 자신에게 맞게 재지정한다.

  4. 사용자를 생성한다.

    CREATE USER ora_user IDENTIFIED BY Risa DEFAULT TABLESPACE MYTS TEMPORARY TABLESPACE TEMP;

    • User = 유저명/ IDENTIFIED BY = 비밀번호

  • User를 삭제해야 할 경우

    drop user 사용자이름;

  • 이미 테이블을 만든 경우

    drop user 사용자이름 cascade;

  1. 만든 사용자에 권한을 부여한다.

    GRANT DBA TO ora_user;

    • DBA 라는 권한을 부여한다.

  2. 만든 사용자로 접속한다.

    connect 유저명/비밀번호

    • 로그인한 사용자 이름 확인

      select user from dual;

오라클 삭제

  1. 오라클 서비스를 중지시킨다.

    • 제어판 - 시스템 및 보안 - 관리도구 - 서비스

    • Oracle로 시작하는 모든 서비스를 중지시킨다.

  2. 처음 압축 해제 했던 설치 파일 폴더의 deinstall 경로에 들어가서 deinstall.bat을 관리자 권한으로 실행한다.
