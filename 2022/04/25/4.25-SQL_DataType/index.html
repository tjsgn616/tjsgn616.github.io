SQL - 테이블과 데이터 타입

테이블

  • 테이블: 데이터를 담고 있는 객체

    • 테이블 명, 컬럼 명의 최대 크기는 30 바이트이다.
    • 테이블 명, 컬럼 명으로 예약어는 사용할 수 없다.(예약어 = 오라클이 미리 선점한 단어)
    • 테이블 명, 컬럼명으로 문자 ,숫자, ‘_’, ‘$’, ‘#’ 을 사용할 수 있지만 첫글자는 문자여야만 한다.
    • 한 테이블에 사용 가능한 컬럼은 최대 255개이다.

  • 테이블 생성( CREATE TABLE 테이블명(조건) )

    • 쿼리를 작성 후, 페이지 전체 명령어 실행은 F5
    • 한 구역의 명령어만 실행은 F9 실행

데이터

  • 데이터 추가: INSERT INTO () VALUES ();

  • 데이터 확인: SELECT FROM

데이터 타입

1. 문자 데이터

  • 각각의 COLUMN 에 들어갈 데이터 타입을 설정한 것이다.
    • CHAR(byte or char): 고정길이 문자
      • CHAR은 고정된 크기 할당 시 어떤 문자를 넣어도 고정된 크기를 차지한다.
    • VARCHAR2(byte or char) : 가변길이 문자
      • VARCHAR2는 넣는 문자의 크기에 따라 다른 크기를 가지며 최대치를 지정하는 것이다.
      • 공간 효율화를 위해 VARCHAR2를 권장한다.
    • NCHAR(byte or char) : 고정길이 유니코드 문자
    • NVARCHAR2(byte or char) : 가변길이 유니코드 문자

2. 숫자 데이터

  • 오라클에서 제공하는 숫자형 데이터 이다.

    • NUMBER(p, s) : 가변 숫자 p(138, 디폴트 = 38)와 s(-84127, 디폴트=0)는 십진수 기준
    • FLOAT(p): NUMBER의 하위 타입, p(1~128, 디폴트 = 128)는 이진수 기준
    • BINARY_FLOAT : 32비트 부동 소수점 수
    • BINARY_DOUBLE : 64비트 부동 소수점 수

  • 예시)

    • 출력: 1234567891000, 1234567891234
    • FLOAT(32)의 경우 FLOAT 뒤의 숫자는 2진수 기준 자릿수 이기 때문에, 2진수 32자릿수는 10진수 기준 10자릿수 이므로 10번째 자리 까지인 1234567891000으로 입력된다.
    • FLOAT의 경우 기본값이 126 이기 때문에 10진수 기준 38 자릿수까지 표현할 수 있기 때문에 그대로 출력이 된다.

3. 날짜형 데이터

  • DATE: 연,월,일,시,분,초 까지 입력 가능하다
  • TIMESTAMP: 연,월,일,시,분,초,밀리초 까지 입력 가능하다(DATE 보다 정확도가 조금 더 높다)

4. LOB 데이터 타입

  • LARGE OBJECT의 약자로 대용량 데이터를 저장할 수 있는 데이터 타입이다.
  • 비정형 데이터는 그 크기가 매우 크기 때문에 그러한 데이터를 저장하는데 사용된다.
