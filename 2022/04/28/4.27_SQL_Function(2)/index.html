SQL 함수(2)

A. 변환 함수

  • 서로 다른 유형의 데이터 타입으로 변한해주는 함수
  • 명시적 형 변환을 위한 함수

1. TO_CHAR (숫자 혹은 날짜, format)

  • 숫자나 날짜를 문자로 변환해 주는 함수

  • 매개변수로는 숫자나 날짜가 올 수 있고 format 값에 따라 결과를 특정 형식에 맞게 출력할 수 있다.

    • 결과: 123,456,789

  • 자주 사용하는 포맷

2. TO_NUMBER(expr, format)

  • 문자나 다른 유형의 숫자를 NUMBER 형으로 변환하는 함수
  • 예시)
    • 결과: 123456

3. TO_DATE(char, format), TO_TIMESTAMP(char, format)

  • 문자를 날짜형으로 변환하는 함수이다
  • 이전의 ‘날짜 변환 형식’과 같은 항목이 올 수 있다.
  • TO_DATE 는 DATE 형으로, TO_TIMESTAMP 는 TIMESTAMP 형으로 변환해 값을 반환한다.

B. NULL 관련 함수

  • NULL을 비교할 때는 IS NULL 또는 IS NOT NULL 구문을 사용하는데, 오라클에서는 NULL을 연산 대상으로 처리하는 SQL 함수를 제공하고 있다.

1. NVL(expr1, expr2), NVL2(expr1, expr2, expr3)

  • NVL은 expr1이 NULL 일 때, expr2를 반환한다.
  • NVL2 는 expr1이 NULL이 아니면 expr2를, NULL이면 expr3를 반환한다.
    • 예시)
    • 앞의 쿼리는 commision_pct가 NULL 인 사원은 그냥 급여를, Null이 아닌 사원은 급여 + 급여*커미션을 조회하고있다.

2. COALESCE (expr1, expr2, …)

  • COALESCE 는 매개변수로 들어오는 표현식에서 NULL 이 아닌 첫 번째 표현식을 반환한다.

3. LNNVL(조건식)

  • LNNVL은 매개변수로 들어오는 조건식의 결과가 FALSE나 UNKNOWN이면 TRUE, TRUE 면 FALSE를 반환한다.

    • 예시)

    • NULL 값은 일반적인 조건식으로는 출력되지 않으므로 NVL을 이용해 0값으로 바꿔주었다.

    • 위와 같은 식을 LNNVL 함수를 이용해 똑같이 만들 수 있다.

    • LNNVL 함수는 매개변수로 들어오는 조건의 결과가 TRUE이면 FALSE를, FALSE나 UNKNOWN이면 TRUE를 반환하므로, 조건식을 ‘commission_pct< 0.2’가 아닌 ‘commission_pct>=
       0.2
      ’로 바꾸었다.

4. NULLIF (expr1, expr2)

  • NULLIF 함수는 expr1과 expr2를 비교해 같으면 NULL, 같지 않으면, expr1을 반환한다.

C. 기타 함수

1. GREATEST(expr1, expr2, …) , LEAST(expr1, expr2, …)

  • GREATEST는 매개변수로 들어오는 표현식 중에서 가장 큰 값을, LEAST는 가장 작은 값을 반환한다.
  • 숫자 뿐 아니라 문자도 비교할 수 있다.

2. DECODE (expr, search1, result1, search2, result2, …, deafult)

  • DECODE 는 expr 과 search1을 비교해 두 값이 같으면 result1를, 같지 않으면 search2 와 비교해 두 값이 같으면 result2를 반환하고, 같은 식으로 계속 비교해도 같은 값이 나오지 않으면 default 값을 반환한다.
  • 비교 값과 결과 값을 일일이 나열해야 하므로, CASE표현식을 쓰는것이 더 깔끔하긴 하다.
