그룹 쿼리와 집합 연산자

A. 기본 집계 함수

  • 집계 함수란 대상 데이터를 특정 그룹으로 묶은 다음 이 그룹에 대해 총합, 평균, 최댓값, 최솟값 등을 구하는 함수이다.

1. COUNT(expr)

  • 쿼리 결과 건수(전체 로우 수)를 반환하는 집계 함수이다.
  • 테이블 전체 로우는 물론 WHERE 조건으로 걸러진 로우 수도 반환 할 수 있다.
  • NULL 값은 세지 않는다.
  • COUNT(DISTINCT “”): 고유 값들의 갯수를 셀 수 있다.

2. SUM(expr)

  • SUM은 expr 의 전체 합계를 반환하는 함수이다.
  • expr에는 숫자형만 올 수 있다.

3. AVG(expr)

  • AVG는 expr의 평균값을 반환하는 함수이다.
  • 형태나 쓰임새는 COUNT, SUM과 같다.

4. MIN(expr), MAX(expr)

  • MIN 은 최솟값, MAX는 최댓값을 반환하는 함수이다.

5. VARIANCE(expr), STDDEV(expr)

  • VARIANCE는 분산, STDDEV는 표준편차를 구해서 반환하는 함수이다.

B. GROUP BY 절과 HAVING 절

1. GROUP BY

  • GROUP BY 절은 특정 그룹으로 묶어 데이터를 집계하는데 사용된다.

  • 그룹으로 묶을 컬럼명이나 표현식을 GROUP BY 절에 명시해서 사용하며 GROUP BY 구문은 WHERE 과 ORDER BY 절 사이에 위치한다.

    • 예시)

  • 그룹 쿼리 사용시 SELECT 리스트에 있는 컬럼명이나 표현식 중 집계 함수를 제외한 모두를 GROUP BY 절에 명시해주어야 한다.

    • SQL 오류: ORA-00979: GROUP BY 표현식이 아닙니다.
    • 월 까지 밖에 없는 데이터에서 2013년 11월로 period를 정해주었으므로 굳이 그룹에 포함시킬 필요 없지만, 구문 문법상 GROUP BY 절에 포함 시켜야만 한다.

2. HAVING

  • HAVING 절은 GROUP BY 절 다음에 위치해 GROUP BY 한 결과를 대상으로 다시 필터를 거는 역할을 한다.

    • 예시)

    • 위 쿼리 결과에서 대출 잔액이 100조 이상인 것만 추출하는 조건을 추가 하여 쿼리를 작성한 것이다.

  • 주의할 점은 WHERE 절은 쿼리 전체에 대한 필터 역할을 하고, HAVING 절은 WHERE 조건을 처리한 결과에서 GROUP BY를 수행한 후 산출된 결과에 대해 다시 조건을 걸어 데이터를 걸러내는 것이다.

C. ROLLUP 절과 CUBE 절

  • ROLLUP 과 CUBE 는 GROUP BY 절에 사용되어 그룹별 소계를 추가로 보여주는 역할을 한다.

1. ROLLUP(expr1, expr2, …)

  • ROLLUP은 expr 로 명시한 표현식을 기준으로 집계한 결과(추가적인 집계 정보)를 보여준다.

  • ROLLUP절에 명시할 수 있는 표현식에는 그룹핑 대상, 즉 SELECT 리스트에서 집계 함수를 제외한 컬럼 등의 표현식이 올 수 있다.

  • 명시한 표현식 수와 순서에 따라 레벨별로 집계한 결과가 반환된다.(표현식 개수가 n개면 n+1레벨까지, 하위레벨에서 상위레벨 순으로 데이터가 집계된다)

  • 예시)

    • ROLLUP 미적용

    • ROLLUP 적용

    • ROLLUP 절에 period와 gubun 을 명시하였으므로 총 3레벨이다. 최하위 레벨인 period,gubun , 다음 하위 레벨인 period, 가장 상위레벨인 전체 합계가 집계 되었다.

  • ROLLUP의 또다른 유형으로 분할 ROLLUP이 있다.

    • GROUP BY expr1, ROLLUP(expr2, expr3) 과 같이 진행하면, 위와 같이 전체 3단계가 되지만, 결과는 (expr1, expr2, expr3), (expr1, expr2),(expr1) 별로 집계되고 전체 합계는 집계되지 않는다.

2. CUBE(expr1, expr2, …)

  • CUBE는 ROLLUP과 비슷하지만 조금 다르다.

  • ROLLUP은 레벨별 순차적 집계를 하는 반면 CUBE는 명시한 표현식 갯수에 따라 가능한 모든 조합으로 집계한 결과를 반환한다.

    • 그래서 CUBE는 2^(expr 갯수) 만큼 종류별로 집계된다.

  • 예시)

    • expr 수가 2개 이므로 총 4가지 유형의 집계가 나온다.(전체, gubun, period, period+gubun)

  • CUBE도 분할 CUBE유형이 존재한다.

    • GROUP BY expr1, CUBE(expr2,expr3) 를 하게 되면 (expr1, expr2, expr3), (expr1, expr2), (expr1, expr3), (expr1) 총 4가지 유형으로 집계된다.

ROLLUP 과 CUBE 집계 방식

표현식 집계 종류
ROLLUP(expr1, expr2) expr1 + expr2
expr1
전체
GROUP BY expr1, ROLLUP (expr2, expr3) expr1 + ( expr2 + expr3)
expr1 + ( expr2)
expr1
GROUP BY ROLLUP(expr1), expr2 expr2 + expr1
expr2
CUBE (expr1, expr2) expr1 + expr2
expr1
expr2
전체
GROUP BY expr1, CUBE ( expr2, expr3) expr1 + ( expr2 + expr3)
expr1 + ( expr2)
expr1 + ( expr3)
expr1

B. 집합 연산자

  • 데이터 집합(set)을 대상으로 연산을 수행하는 연산자
  • 집합 개념과 비슷하다.

1. UNION

  • 합집합을 의미한다.
  • 데이터 집합들의 집합 원소를 모두 포함한 결과가 반환된다.
  • 예시)

    • A그룹

    • B그룹

    • UNION 명령어 실행

      • 합집합 개념을 적용해서 겹치는 종류는 추출하지 않는다.
  • UNION ALL 명령어는 합쳐서 전체를 다 추출한다.

2. INTERSECT

  • 교집합 개념을 적용하여 공통된 항목만 추출한다.

3. MINUS

  • 차집합 개념을 적용하여 한 데이터 집합에서 다른 데이터 집합과 공통된 항목을 제외한 결과를 추출한다.

집합 연산자 사용 시 주의사항

  1. 집합 연산자로 연결할 각 SELECT 문의 리스트 개수와 데이터 타입을 일치시켜야 한다.
  • 리스트 개수 오류
    • SQL 오류: ORA-01789: 질의 블록은 부정확한 수의 결과 열을 가지고 있습니다.
  • 데이터 타입 오류
    • SQL 오류: ORA-01790: 대응하는 식과 같은 데이터 유형이어야 합니다.
    • seq와 goods 의 데이터 타입이 달라서 오류가 발생하였다.

  1. ORDER BY 절은 맨 마지막 문장에서만 사용 할 수 있다.

    • SQL 오류: ORA-00933: SQL 명령어가 올바르게 종료되지 않았습니다.
    • ORDER BY를 맨 마지막으로 옮겨 주어야 한다.

  2. BLOB, CLOB, BFILE 타입의 컬럼에서는 집합 연산자를 사용할 수 없다.

  3. UNION, INTERSECT, MINUS 연산자는 LONG 형 컬럼에서 사용 할 수 없다.
