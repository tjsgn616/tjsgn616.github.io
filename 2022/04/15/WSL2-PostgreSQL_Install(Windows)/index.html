WSL2 - PostgreSQL 설치 (Windows)

목표

  • WSL2에서 PostgreSQL을 설치
  • pdAdmin은 Windows에 설치

WSL터미널 업데이트(ubuntu package)

sudo apt update , password 입력

sudo apt-get upgrade

WSL2에서 PostgreSQL 설치

  • PostgreSQL 설치 진행 및 버전 확인

sudo apt install postgresql postgresql-contrib

psql --version

  • Database 접근 가능하도록 postgres를 활성화 한다.

sudo service postgresql start

  • 결과: * Starting PostgreSQL 12 database server

sudo service postgresql status # 활성화 상태 확인

  • 결과: 12/main (port 5432): online

  • 활성화 시킨 Database를 종료 시킨다.

sudo service postgresql stop

sudo service postgresql status

사용자 계정 Password 설정

  • 기본 사용자는 admin, 패스워드를 미리 설정한다.

sudo passwd postgres

  • 희망 Password 입력

pgAdmin 설치

  • pgAdmin 설치: • https://www.pgadmin.org/download/pgadmin-4-windows/
    • 파일 실행시 관리자 권한으로 실행하도록 한다.
    • install mode는 ‘install for all users’를 선택한다.

  • pgAdmin 프로그램 실행 전, postgres를 활성화 시켜야한다.

    • 처음 실행시 나타나fpdlanff는 password는 아까 wsl2에서 설정한 password

  • 서버 설정

    • General Tab - Name : test

    • Connection Tab

      • password에는 아까 입력한 비밀번호를 입력하면 되는데 아마 에러가 뜰 것이다

      • 다시 wsl 화면에서 Password를 설정한다.

      • (Service 활성화 상태에서)

      sudo -u postgres psql -c "ALTER USER postgres PASSWORD '<new-password>';"

      • 자리에 비밀번호를 입력하면 된다.(따옴표 뺴면안된다)

  • 설정 후 재접속 시 정상적인 접근이 가능하다

DB 생성 및 확인

  • DB 생성을 해본 후 pdAdmin 과 WSL2에서 각각 확인해본다.

  • Database에서 우클릭 후, 아래와 같이 순차적으로 클릭한다.( Database-Create-Database)

    • 새로운 데이터 베이스 명은 dataengineering 으로 한다

  • dataengineering - shemas - public - Tables 에서 Tables를 우클릭 후 Create - Table 선택

    • General 탭 - Name: users

    • Column 탭 - 테이블 열 추가

  • 다시 wsl2에서 psql에 접속 후, dataengineering DB와 생성된 테이블을 조회하는 쿼리를 실행한다.

  • sudo -u postgres psql

    • 결과:

  • dataengineering 테이블이 조회되는지 확인한다.

    • postgres=# \l (영어 소문자 l 이다- list)

  • dataengineering DB에 연결 후, 테이블을 조회한다.

    • postgres=# \c dataengineering
