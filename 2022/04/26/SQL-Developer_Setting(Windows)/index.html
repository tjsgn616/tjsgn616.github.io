#SQL, Oracle, Windows

SQL Developer 설정(Windows)

  • SQL Developer 설정 과정을 기록한다.
    • SQL Developer 링크: https://www.oracle.com/tools/downloads/sqldev-downloads.html

  1. 압축 해제한 후 sqldeveloper.exe 를 실행한다.

    • 설정 변수를 import 할지에 대한 창이 뜨지만 설정한 게 따로 없으므로 그냥 넘어가면 된다.

  2. Oracle 접속을 우클릭하여 새 접속 란을 선택한다.

  3. 데이터베이스 접속 선택을 진행한다.

    • ora_user로 만든 후, 설정 해둔 IDENTIFIED BY 로 비밀번호로 설정 한다.

    • 설정 후 테스트 를 해보고 문제가 없다면 저장 버튼을 눌러 좌측에 추가되는 지 확인한다.

    • 오류 발생 시 Oracle의 NET Configuration Assistant를 이용해 포트 설정을 완료 한 후 다시 진행해 본다.

    • 포트 설정 후 아래와 같이 나오면 정상상태이다.

      lsnrctl status

  4. 날짜 형식과 시간 기록 형식을 변경한다.

    • [도구] - [환경설정] - [데이터베이스] - [NLS] 항목의 [날짜 형식]과 [시간 기록 형식]을 변경한다.

      • 날짜 형식 : YYYY/MM/DD

      • 시간 기록 형식 : HH24:MI:SS

      • 앞으로 날짜 형식 데이터를 조회 할 때 YYYY/MM/DD 24/MM/SS 형식으로 출력된다.

샘플 스키마 설치

  • 내용 학습을 위한 샘플 스키마를 설치한다.
    • 스키마: 사용자와 데이터 베이스를 구서하는 객체 및 데이터를 포괄하는 개념

  1. 학습하는 책에 맞는 수정된 스키마를 추가 설치한다.

    • 링크: https://github.com/gilbutITbook/006696/tree/master/01장 환경설정
      • expall.dmp 와 expcust.dmp를 다운로드한다.

  2. C 드라이브에 backup 폴더를 생성하여 두 파일을 이동시킨다.

  3. cmd를 실행하여 C:\backup 폴더로 이동한다.

  4. 두 명령어를 실행한다. (사용자명/ 비밀번호 확인)

    imp ora_user/Risa file=expall.dmp log=empall.log ignore=y grants=y rows=y indexes=y full=y

    imp ora_user/Risa file=expcust.dmp log=expcust.log ignore=y grants=y rows=y indexes=y full=y

  5. SQL Developer에서 ora_user의 테이블에 들어가 아래 명령어를 실행한다.
    SELECT table_name FROM user_tables;

  6. 아래에 (4) 에서 불러왔던 파일 속 목록 10개가 나타난다면 완료된 것이다.
