Oracle - SQL 문장(DDL,DML)

2022-04-26

#SQL, Oracle

SQL 문장

  • 객체를 생성하는 DDL(CREATE, ALTER, DROP)과 데이터를 수정하고 만드는 DML(SELECT, INSERT, UPDATE, DELETE, MERGE, 등)이 존재한다.

DDL

  • 객체를 생성하고 제거하는 명령어들

  • CREATE : 테이블,인덱스, 뷰 등의 데이터베이스 객체를 생성하는 명령문

    • 테이블 생성: CREATE TABLE 테이블명(컬럼명1 컬럼1_데이터타입 [NULL, NOT NULL], 컬럼명2 컬럼2_데이터타입 [NULL, NOT NULL], ...)

  • ALTER: 이미 생성된 데이터베이스 객체의 컬럼명, 데이터 타입 등을 변경하는 명령문

    • 컬럼 명 변경: ALTER TABLE 테이블명 RENAME COLUMN 컬럼명1 TO 컬럼명2;
    • 컬럼 타입 변경: ALTER TABLE 테이블명 MODIFY 컬럼명 테이터타입;
    • 컬럼 추가: ALTER TABLE 테이블명 ADD 컬럼명 데이터타입;
    • 컬럼 삭제: ALTER TABLE 테이블명 DROP COLUMN 컬럼명 ;
    • 제약조건 추가: ALTER TABLE 테이블명 ADD CONSTRAINTS 제약조건명 PRIMARY KEY (컬럼명);
    • 제약조건 삭제: ALTER TABLE 테이블명 DROP CONSTRAINTS 제약조건 명;

  • DROP : 생성된 데이터베이스 객체를 삭제하는 명령문

    • 테이블 삭제: DROP TABLE 테이블명;

  • TRUNCATE : 한번 실행 시, 데이터 바로 삭제

    • 명령문은 DROP과 유사하다.
    • Delete 문은 데이터 삭제 후, DB에서도 제거하려면 Commit 할 필요가 있으나 Truncate는 바로 삭제된다.
    • 바로 제거 되기 때문에 Rollback 도 적용 할 수 없다.

  • DESC: 테이블에 있는 컬럼 내역들을 바로 나타내 준다.

    • 컬럼 내역 보기: DESC 테이블명;

    • group by를 통해 사원번호의 중복을 제거하였다.

    2. ex3_3 테이블에 있는 사원의 사번과 일치하면 보너스 금액에 자신의 급여의 1%를 보너스로 갱

  • Commit : 변경한 데이터를 db에 마지막으로 반영

    • cmd 실행 - sqlplus 입력 - id/비밀번호 입력(ora_user/Risa) 입력 - 데이터베이스 접속 완료

    • select * from ex3_4;

    • Commit 명령어로 db에 최종 반영을 해주지 않았기 때문.

    • Commit; 명령어 실행

  • Rollback : 변경한 데이터를 변경하기 이전으로 되돌림
