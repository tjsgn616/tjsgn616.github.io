WSL2- Apache-Airflow 설치와 설정(Windows10)

2022-04-18

#WSL2, Windows10, Apache, Airflow

WSL2에서 airflow 설치하기

  • WSL 환경에서 airflow를 설치하는 과정을 기록하였다.

Step.1 Install pip on WSL

  • airflow를 설치하기 위해 pip를 설치한다.

    sudo apt-get update && sudo apt-get upgrade

    sudo apt install python3-pip

    [sudo] password for username:(비밀번호 입력)

Step.2 Install virtualenv package

  • virtualenv 라이브러리를 설치한다.

    sudo pip3 install virtualenv

    • 기본적인 명령어들: https://staticclass.tistory.com/66, https://www.manualfactory.net/10805 , https://blockdmask.tistory.com/25
    • 직접 명령어 정리한 링크: https://tjsgn616.github.io/2022/04/17/WSL(Linux)-Basic_Vi_Command/
    • clear = 콘솔에 적힌 텍스트 전부 삭제
    • ls = 현재 위치의 디렉토리 목록
    • cd = 디렉토리 변경, cd .. = 상위 디렉토리로 이동
    • mkdir 폴더이름 : 폴더 만든다.

Step 3. Create a virtual evironment

  • C 드라이브에 airflow-test폴더를 생성한다.

  • 해당 디렉터리로 이동한다.

  • 가상 환경을 생성한다.

    virtualenv venv

  • 가상환경에 접속한다.

    source venv/bin/activate

  • .bashrc 파일을 수정한다.

    vi ~/.bashrc

    • vi 명령어 설명

      https://wiki.kldp.org/KoreanDoc/html/Vim_Guide-KLDP/Vim_Guide-KLDP.html

    • i : 입력, :w 저장, :q 입력 종료후 나가기, :wq 저장후 나가기

  • 파일을 열고, 다음과 같은 코드를 추가한다.

    export AIRFLOW_HOME=/mnt/c/airflow-test

  • 파일을 닫을 때는 ESC 누르고 :wq 명령어를 입력하면 된다.

  • 수정된 코드를 업데이트 하기 위해서는 아래와 같이 반영한다.

    source ~/.bashrc

  • 실제로 코드가 반영되었는지 확인하기 위해서는 다음과 같이 확인해본다.

    echo $AIRFLOW_HOME

    • 결과: /mnt/c/airflow-test

Step 4. Apache Airflow 설치

  • PostgreSQL, Slack, Celery 패키지를 동시에 설치하는 코드 입력

    pip3 install 'apache-airflow[postgres, slack, celery]'

  • 에어플로 실행 위해 DB 초기화를 해줘야 한다.

    airflow db init

    • username을 추가하도록 한다.

  • 실제로 잘 구현이 되었는지 확인하기 위해 webserver를 실행한다.

    airflow webserver -p 8081 → 8081 포트가 사용중이라면 다른걸 써야할지도

  • 다음으로 일정 주기로 데이터 흐름이 실행되게 하려면 Scheduler가 필요하다.

    airflow scheduler → 없어도 되는데?

    • 그 후, http://localhost:8081/login/ 로 들어간다.
    • 로그인 화면에서 로그인을 진행하면 정상적으로 다양한 DAGs 파일이 나타난 것을 확인할 수 있다.

Step 5. Default 예제 제거

  • 처음에 나오는 load_examples 들 없애기

  • airflow.cfg 파일을 열고 load_examples = True를 load_examples =True 로 바꿔준다.

    • :/원하는검색단어 ⇒ 단어 검색, n= 다음 결과로 찾아가기

  • 저장 후,

  • 다시 webserver 실행
  • 이제 아무런 예제가 떠오르지 않는다.
