Oracle - SQL 연산자

2022-04-27

#SQL, Oracle

SQL 연산자

수식 연산자

  • ‘ + ’, ‘ - ’ , ‘ * ’ , ‘ / ’
  • +,- 는 단일로 사용될 때 음수와 양수를 표현하는데 사용 가능
  • +,-,*,/ : 이항 연산자로 덧셈, 뺄셈, 곱셈, 나눗셈 연산자로 사용 가능

문자 연산자

  • || 연산자 : 두 문자를 연결하는 연산을 수행한다.

  • SELECT A || 'b' || B AS C : A와 B를 b를 추가한 상태로 C 칼럼으로 표시한다.

  • 예시)

논리 연산자

  • , <, >=, <=, =, <>, !=, ^=

  • 논리연산에서 사용되는 연산자 이다.
  • 값을 비교할 때 숫자 뿐 아니라 문자의 날짜형도 비교 가능하다.

표현식

  • 표현식: 한 개 이상의 값과 연산자, SQL 함수 등이 결합된 식이다.
  • CASE 표현식

    • CASE WHEN 조건 THEN '출력' / ELSE / END

    • ‘출력’ 에 들어가는 출력값들의 데이터 타입은 모두 동일해야만 한다. (’A등급’,’B등급’ = 가능, ‘A등급’, 2 = 불가능

    • 예시)

조건식

비교 조건식

  • ANY, SOME, ALL 키워드로 비교하는 조건식
  • SOME, ANY : 하나라도 만족하는 모든 것을 출력한다.
    • OR 조건문과 비슷하다.
  • ALL : 모든 조건을 동시에 만족해야 한다.
    • AND 조건문과 비슷하다.

논리 조건식

  • AND, OR, NOT 을 사용하는 조건식

  • OR : 여러 조건 중 하나만 만족해도 TRUE를 반환한다.

  • AND : 모든 조건을 만족해야 TRUE를 반환한다.

  • 예시)

  • 예시를 다음과 같이 동일하게 표현할 수 있다.

BETWEEN AND 조건식

  • BETWEEN 은 범위에 해당하는 값을 찾을 때, 크거나 같고, 작거나 같은 것을 찾을 때 사용한다.

  • = 또는 <= 로 변환 가능하다.

  • 예시)

IN 조건식

  • 명시된 값이 포함된 것을 반환하는데 ANY와 비슷한 역할을 한다.

  • NOT IN 을 사용하여 아닌 것을 조회 할 수 있다.(NOT IN ~ : ~ 값을 제외한 전체)

  • IN 예시)

  • NOT IN 예시)

EXISTS 조건식

  • IN 조건식과 비슷하지만 조건절로 값의 리스트가 아닌 서브쿼리만이 올 수 있다. 또한 서브쿼리에서 조인 조건이 있어야만 한다.
  • 예시)
    • WHERE a.department_id = b.department_id 가 서브쿼리의 조인 조건이다.

LIKE 조건식

  • 문자열의 패턴을 검색할 때 사용하는 조건식이다.
  • 예시)

    • 사원이름이 ‘A’로 시작되는 사원을 조회하는 쿼리

    • %는 모든 글자, _는 한 글자만 검색한다.

    • ‘A%’ = AB, ABC 출력가능, ‘A_’ = AB만 출력가능
