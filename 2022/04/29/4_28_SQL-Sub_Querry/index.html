SQL - 서브쿼리

서브 쿼리

  • 서브쿼리 : SQL 문장 안에서 보조로 사용되는 또 다른 SELECT 문
  • 메인 쿼리를 제외한 나머지 모든 SELECT 문은 서브 쿼리다.
  • SELECT, FROM, WHERE, INSERT, UPDATE, MERGE, DELETE 모두에서 사용할 수있다.
  • 서브 쿼리는 특성과 형태에 따라 구분 가능
    • 메인 쿼리와의 연관성에 따라
      • 연관성 없는 서브 쿼리
      • 연관성 있는 서브 쿼리
    • 형태에 따라
      • 일반 서브 쿼리(SELECT 절)
      • 인라인 뷰(FROM 절)
      • 중첩 쿼리(WHERE 절)

1. 연관성 없는 서브 쿼리

  • 메인 쿼리와의 연관성이 없는 서브쿼리(메인 테이블과 조인 조건이 걸리지 않는 서브쿼리)
  • 예시)
    • 메인 쿼리와 서브 쿼리 모두 사원 테이블을 조회하고 있지만, 메인 쿼리와 서브 쿼리 사이의 연관성은 없음.
  • 예시 2)
    • job_history 테이블에 있는 employee_id, job_id 두 값과 같은 건을 사원 테이블에서 찾는 쿼리로 서브 쿼리는 메인 쿼리와 연관성이 없다.
    • 동시에 두개 이상 컬럼 값이 같은 건을 찾고있다.(IN 앞에 있는 컬럼과 서브 쿼리에서 반환하는 컬럼의 개수와 데이터 타입은 같아야한다.)

2. 연관성이 있는 서브 쿼리

  • 메인 쿼리와의 연관성이 있는 서브 쿼리(메인 테이블과 조인 조건이 걸린 서브쿼리)
  • 예시)
    • 서브 쿼리 안에서 메인 쿼리에서 사용된 부서 테이블의 부서 번호와 job_history 테이블의 부서번호가 같은 건을 조회한다.
    • EXISTS 연산자를 통해 서브 쿼리 내에 조인 조건이 포함돼 있다.
    • 결과는 job_history 테이블에 있는 부서만 조회됨
  • SELECT 절에 자체에도 여러개의 서브 쿼리를 넣을 수 있다.

    • 예시)

    • job_history에는 employee_id, department_id 밖에 존재하지 않으므로, employee_name과 department_name을 가져오기 위해 서브쿼리를 SELECT 절에서 사용하였고, 서브 쿼리 안의 WHERE 절에 조건을 추가한 형태이다.

  • 서브 쿼리는 여러 단계에 걸쳐 사용이 가능하다.(중첩 서브쿼리)
  • 연관성 있는 서브 쿼리 또한 UPDATE, MERGE, DELETE 문에서 사용할 수 있다.

3. 인라인 뷰

  • FROM 절에 사용하는 서브 쿼리를 인라인 뷰 라고 한다.
  • 원래 FROM 절에는 테이블 or 뷰 가 오지만, 서브 쿼리를 하나의 테이블이나 뷰 처럼 사용할 수 있다.
  • 예시)
    • 기획부 산하 평균 급여를 구하는 부분을 서브쿼리로 작성하여 FROM 절에 위치시켰다.

복잡한 쿼리를 작성하는 Tip.

  • 복잡한 쿼리를 작성할 때, 작은 단위로 분할한 뒤, 다시 합치는 방식을 이용한다.

  1. 최종적으로 조회되는 결과 항목을 정의한다.

  2. 필요한 테이블과 컬럼을 파악한다.

    → 메인 쿼리 작성

  3. 작은단위로 분할해서 쿼리를 작성한다.

    → 서브 쿼리 작성

  4. 분할한 단위의 쿼리를 하나로 합쳐 최종 결과를 산출한다.

    → 서브 쿼리 작성 후 합치기

  5. 결과를 확인한다.
