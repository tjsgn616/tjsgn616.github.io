Oracle - PL/SQL

2022-05-02

#SQL, Oracle, PL/SQL

PL/SQL

  • PL/SQL이란 집합적 언어의 특징과 계층적 언어의 특징을 모두 지니고 있다.(SQL을 확장한 계층적 언어)
  • 복잡한 비즈니스 로직을 처리할 때(프로그래밍) 유용하게 사용할 수 있다.
  • 블록 단위의 실행을 제공한다.(BEGIN 과 END; 로 구분, 마지막 라인에 /를 입력하면 해당 블록이 실행된다.)
  • !!!!!!!!!!한 줄을 끝낼 때 마다 ; 를 붙여줘야한다 !!!!!!!!
  • 좌측 변수에 우측 값을 입력할 때는 := 를 사용한다.(python의 = 과 같은 역할)

기본 PL/SQL 블록 구조

설정(SET)

  • 한 번 적용하면 SQL을 종료 할 때 까지 그 이후 실행하는 모든 명령문에 적용된다.
    • SET TIMING : 명령문 실행 경과 시간을 함께 출력한다.
    • SET SERVEROUTPUT : 명령문 실행 결과를 출력한다.
      • SET SERVEROUTPUT ON 을 했다면 DBMS_OUTPUT.PUT_LINE(출력하고자하는 값); 을 통해 해당 값을 출력 시킬 수 있다.
      • ON / OFF 를 붙여서 켜고 끄고 할 수 있다.(디폴트 값은 OFF)

DECLARE(선언부)

  • DECLARE는 PL/SQL에서 사용하는 모든 변수나 상수를 선언하는 부분이다.
  • 변수, 상수, 커서 등을 선언한다.
    • DECLARE 변수이름 변수타입 으로 진행한다.
    • 예) DECLARE vi_num NUMBER;
  • 변수 타입을 지정할 때 명시적으로 지정하지 않아도 된다.
    • %ROWTYPE : 해당 테이블이나 뷰의 컬럼 속성을 그대로 가져온다. ( 변수명 테이블이름%ROWTYPE)
    • %TYPE : 해당 테이블의 컬럼 속성을 지정하여 그대로 가져온다.( 변수명 테이블이름.컬럼명%TYPE )

BEGIN(실행부)

  • BEGIN 은 절차적 형식으로 SQL문을 실행할 수 있도록 로직을 기술할 수 있는 부분이다.(실질적 실행 코드 작성하는 곳)
  • 제어문, 반복문, 함수 등을 사용 가능하다.

EXCEPTION(예외 처리부)

  • EXCEPTION 은 PL/SQL문이 실행되는 중 에러가 발생 했을 때, 이를 해결하기 위한 문장을 기술 할 수 있는 부분이다.

END(실행문 종료)

  • END는 실행문을 종료한다.

PL/SQL 프로그램의 작성 요령

  • 한 문장이 끝날 때 마다 ;(세미콜론)을 붙여서 한 문장이 끝났다는 것을 명시 해 주어야한다.
  • END의 뒤에 ;(세미콜론)을 붙여서 한 블록이 끝났다는 것을 명시 해 주어야 한다.
  • PL/SQL 전체 블록의 마지막에 / 를 붙여서 종결 시켜 주어야한다.

ACCEPT PROMT

  • 변수에 변수를 직접 입력하게 할 수 있는 명령문

    • ACCEPT 변수명 PROMT ‘입력창에 나올 문장’

      • 예시) accept p_num1 prompt '첫번째 숫자를 입력하세요’

    • BEGIN 문에서 해당 변수를 사용 하고자 하면 ‘&’를 붙여서 사용한다.

      • 예시) v_sum = &p_num1 + &p_num2 + &p_num3
